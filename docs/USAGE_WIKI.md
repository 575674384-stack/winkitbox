# WinKitBox 使用 WIKI

## 快速开始

- 在设置页确认工具目录。便携工具、ZIP 工具和收集型工具会优先放在这个目录下，后续迁移电脑时也更容易备份。
- 如果要使用 AI 添加、AI 推荐或 AI 修复，请在设置页保存 OpenAI 兼容接口、API Key 和模型名，并点击测试连通性确认可用。
- 进入查看本机页面，先刷新系统信息，再查看 Windows 环境体检结果；能一键修复的项目会显示修复入口。

## 选择和安装工具

- 勾选工具卡右上角复选框即可加入安装计划，右侧安装计划会显示当前选中数量和执行进度。
- 点击顶部统计卡的已选择、已安装等状态可以快速筛选，再次点击相同统计卡会取消筛选。
- 已选择工具可以批量移动到目标分类；自定义分类支持增删改和排序，全部工具与自定义添加为固定分类。
- 安装、重装、卸载会记录到日志中心。失败时可从工具卡或日志详情进入 AI 修复。

## 添加自定义工具

- 本地文件支持 exe、msi、zip、lnk、bat、cmd、ps1。选择文件后可直接添加，也可以让 AI 分析安装和启动方式。
- 链接添加支持 GitHub、官网、下载页等地址。AI 会尝试生成结构化安装方案，确认后加入工具箱。
- 手动命令适合只加入工具箱、不负责安装，或者需要自定义启动命令的场景。
- 自定义工具默认进入自定义添加分类，也可以在添加前选择其他分类。

## GitHub 榜单和 AI 助手

- GitHub 周榜和月榜会优先展示 Windows 应用相关项目，项目简介会自动翻译为中文。
- 点击项目卡片上的去添加会直接调用 AI 分析并加入工具箱，默认进入自定义添加分类。
- AI 助手可以按自然语言需求推荐多个 Windows 开源项目，适合不知道具体软件名时使用。
- 如果 GitHub 访问不稳定，可以在设置页配置代理模式或 GitHub Token。

## 工具更新中心

- 只检测更新会读取 winget、GitHub Release 或下载源信息，不会执行升级命令。
- 检测到可更新后，手动点击更新或批量更新才会执行对应升级或重装流程。
- 来源健康检测用于发现下载链接、Release 匹配或 winget 包名异常，异常项可以尝试 AI 修复。
- 不想处理的版本可以忽略，忽略记录会影响当前更新列表展示。

## 日志中心

- 操作历史保存安装、卸载、更新、启动、环境修复等动作，包含时间、状态、摘要和失败输出。
- 实时输出显示当前会话里的 PowerShell 输出，适合观察正在执行的任务。
- AI 日志专门保存 AI 推荐、AI 添加和 AI 修复的详细回复，便于复查模型给出的判断。
- 日志可以按状态、类型、工具名和关键词筛选，也可以导出 JSON 或 TXT。

## 查看本机和环境体检

- 刷新配置后可查看系统版本、CPU、内存、硬盘、显卡、网卡和 IP/DNS 信息。
- IP/DNS 调整会通过提权 PowerShell 执行，执行前会出现确认弹窗。
- 公共 DNS 推荐可以测试延迟，测试域名可选择预设或手动输入。
- Windows 环境体检包含 winget、PowerShell、.NET、VC++、WebView2、长路径和 UTF-8 beta 等项目。

## 备份、主题和安全边界

- 导出配置会保存工具目录、分类、自定义工具、主题和 AI 设置摘要；文件大小超过 1MB 会被拒绝导入。
- 主题背景、玻璃参数和自定义背景保存在本机设置里，更新应用不会清空用户自定义工具。
- AI 输出不能直接变成任意 PowerShell。系统只接受受控的 winget、安装包、便携包或本地命令形态。
- 不要把 GitHub Token、API Key、私有下载地址或本机证书提交到仓库。
