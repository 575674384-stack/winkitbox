## Tasks

1. Asset/theme generation and theme contract
   - Generate three new full-screen anime-style themes: light purple fantasy, light red elegant, ninja/anime orange style.
   - Generate reusable theme-aware topbar/stat decorative assets if needed.
   - Update `src/core/themes.ts`, `src/core/themes.test.ts`, CSS variables and theme picker.

2. UI cleanup and typography
   - Change brand version line to `Version: vX.Y.Z`.
   - Remove left-bottom developer credit UI and unused avatar import/asset reference.
   - Update app font stack for higher quality while keeping tool-card body/description in serif/Songti-style font.
   - Make command preview white background / dark text and equalize install/uninstall action sizing.
   - Remove right-panel manual source block.

3. Source health repair UX
   - Rewrite source-health copy to plain language.
   - Add repair buttons for abnormal source rows and update rows.
   - Use existing AI repair flow where applicable and add direct recheck/log actions.
   - Make source-health decorative image theme-aware or neutral, not hard-coded mint.

4. Automatic config snapshots
   - Add startup snapshot logic that creates a config snapshot on each app launch while existing retention keeps only recent snapshots.
   - Keep manual backup UI.
   - Add pure helper tests for backup retention/snapshot naming if possible.

5. GPU detection repair
   - Filter virtual/display-only adapters from Win32_VideoController results.
   - Prefer real GPU names and calculate VRAM from AdapterRAM or fallback PNP/registry if available.
   - Avoid showing virtual adapters such as Zako, USB Mobile Monitor Virtual Display, GameViewer Virtual Display Adapter.

6. Tool page AI recommendations
   - Add right-panel AI recommendation section based on selected tools.
   - Reuse saved AI settings and structured response validation.
   - Return up to 5 items with name, short description, GitHub URL, and add action.
   - Allow previewing GitHub before adding.

7. Notes page
   - Add navigation item before Log Center.
   - Add persistent local notes model and settings storage/IPC or localStorage, multiple note cards, editor, create/rename/delete.
   - Keep UI similar density to tool cards.

8. Documentation and release
   - Update README and AGENTS release notes for v0.6.2.
   - Run tests/build/package.
   - Commit, push, create GitHub Release, upload Setup and Portable.

## Risks

- `src/App.tsx` is already large; prefer extracting core helpers and keeping UI additions scoped.
- Image assets can bloat package size; compress/copy only final selected images.
- Source repair must not silently execute AI or modify tools without user action.
- GPU memory reporting can vary by driver; show filtered real adapters and best-effort memory rather than false precision.
