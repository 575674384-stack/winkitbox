# WinKitBox v0.7.1 UI, Theme, Performance, and Release Plan

## Goal

Fix the v0.6.2 regressions and ship v0.7.1:

- replace native Windows confirmation prompts with an app-styled confirmation modal;
- fix the three new themes not persisting/effectively applying;
- unify top bars, stats cards, empty/error states, and page visual density;
- add bounded long-list optimization for the tool grid;
- use image-generated per-page topbar artwork rather than reusing the active theme image;
- redesign Notes as card-first notebooks with modal editing and no permanent right editor;
- make the font change visibly affect the UI while keeping tool-card descriptions serif;
- fix RustDesk source health by replacing the stale winget source with a live GitHub Release source;
- let the task center auto-hide when all tasks finish, while still allowing explicit close;
- split App/main/styles into smaller owned modules where practical;
- bump, build, package, push, and publish GitHub Release v0.7.1.

## Architecture

Keep the existing Electron + React ownership model. Move reusable UI/presentation logic out of `src/App.tsx` and `src/styles.css` without changing public IPC contracts unnecessarily.

## Tech Stack

- Electron main process: CommonJS.
- Renderer: React + TypeScript + Vite.
- Tests: Vitest + jsdom.
- Packaging: electron-builder.
- Images: built-in image generation, copied into `assets/topbars/`.

## Plan Basis

Facts:

- `electron/main.cjs` still only allows `azure`, `mint`, and `amber` theme IDs.
- RustDesk's current `wingetId: RustDesk.RustDesk` is not found by `winget show` or `winget search` in the current environment.
- `src/App.tsx` contains native `window.confirm` calls.
- Notes currently has a card list plus a sticky right-side editor, creating duplicated creation/editing affordances.
- `src/App.tsx`, `electron/main.cjs`, and `src/styles.css` are all large and need bounded extraction.

Assumptions:

- A bounded grid virtualization implementation is sufficient for current desktop minimum size; no external virtual-scroll dependency is needed.
- The topbar images can be generated as one atlas and cropped into separate project assets.

Unknowns:

- The user's exact local RustDesk source-health failure message. Current winget reproduction is enough to justify changing the built-in source.

## Risks

- UI smoke may reveal hidden text overlap due new topbar images.
- App extraction from a large file can introduce prop wiring mistakes.
- GitHub Release source-health can fail if GitHub is blocked, but it is a more current source than the removed winget package.

## Retirement

- Retire native `window.confirm` usage from renderer.
- Retire stale RustDesk winget source.
- Retire theme ID duplication between renderer and main.
- Retire permanent notes right editor.
- Retire topbar reuse of active theme image for page header artwork.
