# WinKitBox v0.6.1 Toolbox Polish Plan

Goal: ship v0.6.1 with the requested feature upgrades except CI automation, plus sidebar/settings visual polish, developer avatar/GitHub identity UI, version-label cleanup, README sync, packaging, push, and GitHub Release.

Architecture: keep Electron main process as the owner of privileged/network probes; keep React renderer as the owner of user workflow and display; keep `src/core/*` as typed shared behavior and testable pure logic.

Tech Stack: Electron, React, TypeScript, Vite, Vitest, electron-builder.

Baseline/Authority Refs:
- `AGENTS.md`
- `docs/aegis/baseline.md`
- User objective for v0.6.1

Compatibility Boundary:
- Do not add CI automation.
- Keep AI-generated tools constrained to existing structured install shapes.
- Keep environment/system changes user-triggered.
- Do not commit generated packages, logs, private certs, or tokens.
- Do not break existing import/export config.

Verification:
- Targeted tests for new core helpers.
- `npm test`
- `npm run build`
- `npm run package:win`
- GitHub Release `v0.6.1` with Setup + Portable assets.

## Scope Tasks

1. Tool source health check:
   - Add source-health data model and descriptors.
   - Add Electron IPC for user-triggered source checks.
   - Surface source-health panel in Tool Update Center.

2. Update Center strategy clarity:
   - Add strategy labels/descriptions.
   - Show each tool as winget precise update, GitHub/reinstall refresh, manual, none, or unknown.

3. Windows environment repair clarity:
   - Improve health panel copy and repair affordances without creating a duplicate environment dependency center.
   - Keep repair actions in the existing system page.

4. Execution experience:
   - Add clearer command/detail affordances in task center/detail surfaces.
   - Add task retry and user-visible timeout/risk messaging where supported by existing task model.

5. Add Tool simplification:
   - Present local/link/manual add as clearer guided choices.
   - Keep advanced fields collapsible.

6. Catalog search and batch management:
   - Add UI for source/risk/update filters where low-risk.
   - Add batch selected-tool category move and clear selection affordances.

7. Download safety:
   - Add checksum/hash fields to tool shape and source-health reporting.
   - Compute or display hash status for supported direct/release sources.

8. Config backup and rollback:
   - Add local config snapshot helpers and IPC.
   - Add settings UI for creating/restoring recent config backups.

9. README sync:
   - Update user-facing README for current GitHub AI add behavior and v0.6.1 features.

10. Visual polish and identity:
   - Beautify left nav group headers and settings/sidebar red-box areas.
   - Add developer avatar asset from the provided image.
   - Add `GitHub` black icon + `Dev by <avatar>` UI.
   - Remove “装机工具箱” from the top-left version line.
   - Generate optional anime-style decorative asset if useful and save it into project assets.

## Risk Notes

- `src/App.tsx` is large; prefer extracting pure helpers to `src/core/*` where possible.
- Electron IPC changes require synchronized preload and TypeScript declaration updates.
- Source-health checks can be slow; keep them user-triggered and bounded.
- Release should include existing Aegis docs changes unless the user asks otherwise.

## Implementation Notes

- Added `src/core/toolSourceHealth.ts` and Electron `check-tool-sources` IPC for bounded, user-triggered source checks.
- Added `src/core/configBackups.ts` and Electron backup IPC for recent settings snapshots.
- Updated Tool Update Center, catalog batch tools, task center, Add Tool tabs, environment copy, sidebar identity, README, and release notes.
- Verification target remains full tests, production build, Windows package, and GitHub Release `v0.6.1`.
