# Work Records

This directory contains per-task work records for medium or high complexity work in WinKitBox. Each record lives under `docs/aegis/work/YYYY-MM-DD-<slug>/` and follows the Aegis long-task continuation protocol.

## Layout

| File | Purpose |
|------|---------|
| `10-intent.md` | Task intent, scope, non-goals, risks, baseline read-set. |
| `20-checkpoint.md` | Latest checkpoint: completed, in-progress, blocked, next step, drift check. |
| `90-evidence.md` | Commands, logs, screenshots, or manual checks that support completion claims. |
| `99-reflection.md` | Post-completion reflection: what worked, what changed, follow-up risks. |

## When to Create a Work Record

- The task has multiple phases or may span sessions.
- The task touches architecture, contracts, shared workflows, or verification gates.
- A durable trace of intent, evidence, and drift is useful for handoff or review.
